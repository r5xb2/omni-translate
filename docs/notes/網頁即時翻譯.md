## 1. 執行摘要 (Executive Summary)

### 1.1 專案概述
OmniTranslate-GROQ 是一款專為跨國技術會議設計的網頁版即時翻譯工具。該系統利用 GROQ 基礎設施提供的超高速推理能力，整合 Whisper 語音轉文字（STT）與 GPT-OSS-120B 大語言模型，解決實體會議中外語溝通的延遲與技術術語誤解痛點。系統強調「隱私優先（Privacy-First）」，所有敏感資訊如 API Key 均存於客戶端。

### 1.2 商業目標
*   **溝通效率提升**：降低跨語言會議中 40% 的溝通停頓時間。
*   **專業度準確化**：透過技術 Domain-Specific Prompt，將 SSD、Semiconductor 等領域的翻譯準確率提升至 95% 以上。
*   **成本優化**：利用 GROQ LPU 降低單次會議的翻譯成本，相較於傳統翻譯官或低速 API 具備更高性價比。

### 1.3 專案範圍摘要
*   **包含**：麥克風收音、自動語音偵測（VAD）、GROQ 雙模型串接、上下文關聯翻譯、英中對照 UI、Local Storage 安全儲存、.txt/.md 導出。
*   **不包含**：多語系同時翻譯（首版僅限英翻中）、雲端錄音存檔、多人協作編輯。

## 2. 專案背景與動機

### 2.1 業務背景
在 ICT（資通訊）產業中，與國外廠商（如晶片商、系統廠）的實體會議頻繁。現有的翻譯工具（如 Google Translate）無法有效處理長篇連續語音，且缺乏對特定技術領域（如 NAND Flash 映射、CPU 指令集）的語境理解，導致決策效率下降。

### 2.2 現況問題分析
*   **延遲問題**：傳統 LLM 推理速度慢，導致「說完話」與「看到翻譯」之間存在 5-10 秒延遲，無法跟上會議節奏。
*   **隱私疑慮**：企業不希望 API Key 或會議摘要存儲在第三方伺服器。
*   **術語誤譯**：通用模型常將技術專有名詞轉換為非產業習慣用語（例如將「Memory」譯為「內存」而非台灣習慣的「記憶體」）。

### 2.3 解決方案概述
本專案透過 GROQ LPU 的亞秒級延遲特性，實現「說完即譯」。並引入「滾動式上下文（Rolling Context）」機制，確保模型在翻譯每一句話時都能參考前 5 句的對話背景。

*   **上下文規則**：當用戶點擊「暫停」再「開始」時，系統必須抓取暫停前的最後 5 句作為起始語境。

## 5. 非功能性需求 (Non-Functional Requirements)

### 5.1 效能需求 (Performance)
| 指標 | 目標值 | 測量方式 |
|------|--------|----------|
| 單句翻譯端到端延遲 | < 1.5 秒 | Chrome DevTools Network Tab |
| VAD 偵測反應 | < 200 ms | Web Audio API 時鐘 |
| 支援連續會議時長 | > 180 分鐘 | 壓力測試（不當機、不記憶體流失） |

### 5.2 安全需求 (Security)
*   **零服務器存儲**：Node.js 後端僅作為 API Proxy（如需解決 CORS），不得記錄任何音訊或文字內容。
*   **前端加密**：API Key 在 LocalStorage 中建議進行簡易 Base64 或更高階加密儲存。

### 5.3 可用性需求 (Availability)
*   **離線緩衝**：若網路暫時中斷，應顯示錯誤提示並嘗試保留未送出的音訊片段。

### 5.4 相容性需求 (Compatibility)
*   **瀏覽器**：Google Chrome 100+, Microsoft Edge 100+（需支援 Web Audio API）。
*   **設備**：實體會議室專用麥克風、筆電內建麥克風。

## 6. 用戶故事與使用案例 (User Stories & Use Cases)

### 用戶故事
| 編號 | 用戶故事 | 優先級 | 估計點數 |
|------|----------|--------|----------|
| US-001 | 作為 PM，我想要系統能自動感應對方說完話，以便我不用分心操作按鈕。 | M | 5 |
| US-002 | 作為工程師，我希望翻譯能精確區分 SSD 的相關術語，以便我不必重新確認細節。 | M | 3 |
| US-003 | 作為法務，我希望 API Key 只存在我的電腦，以便符合公司資安規範。 | M | 2 |

### 關鍵使用案例 (Use Case)
**UC-001: 進行技術會議翻譯**
- **主要參與者**：會議主持人
- **前置條件**：已輸入有效 GROQ API Key 並開啟麥克風權限
- **主要流程**：
  1. 用戶選擇「ICT 技術」Prompt 模板。
  2. 點擊「Start」按鈕。
  3. 系統監聽麥克風，偵測到用戶發言。
  4. 發言結束（靜音 1.5s），系統將音訊送至 Whisper。
  5. 獲取原文後，連同前 5 句上下文送至 GPT-OSS-120B。
  6. 介面表格同步更新原文與中文。
- **替代流程**：環境雜音過大導致 VAD 失效 -> 用戶點擊「Force Submit」手動切分。
- **後置條件**：生成完整的對話清單供導出。
